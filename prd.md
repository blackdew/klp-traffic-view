# PRD — KLP TrafficView 데모 다듬기

> 신호등 인지형 보행 길찾기 앱(팀 K.L.P)을 **포트폴리오/제출용 라이브 데모**로 다듬기 위한 작업 명세
> 작성: 2026-06-19 · 상태: 계획 확정, 구현 대기

---

## 1. 목표와 범위

### 1.1 "완성"의 정의
이 앱은 데모로는 이미 `(완료)` 상태다. 이번 작업의 "완성"은 **제품화가 아니라 데모 다듬기**다 — 현재 동작하는 데모를 클릭 한 번에 체험 가능한 포트폴리오 산출물로 만든다.

| 결정 | 선택 |
|------|------|
| 완성 방향 | 데모 다듬기 (제출·포트폴리오용) |
| 라이브 호스팅 | GitHub Pages |
| 배포 버전 | 소스 버전(빠른 로딩) — 23MB 단일 파일이 아닌 `traffic_view/*.jsx` |

### 1.2 범위 밖 (Non-Goals)
다음은 "제품화/엔지니어링 완성" 방향에 속하므로 이번 작업에서 **의도적으로 제외**한다.

- 실제 지오코딩·경로탐색 (주소 검색 → 좌표 → 실경로 → 실 횡단보도 매핑)
- 단위 테스트 인프라, Vite 등 빌드 파이프라인, CI
- API 키 서버사이드 이전 등 운영 보안 전환

## 2. 현황 진단

### 2.1 이미 완성된 부분
- **타이밍 알고리즘 (두뇌)**: `signalPhase` / `greenWindows` / `recommendSpeed` 순수함수로 구현 완료. 초록불 구간 역산 → 권장 보행속도 산출.
- **실데이터 연동 + 폴백**: data.go.kr KLID 신호잔여시간 API 연동, Cloudflare Worker CORS 프록시, 실패 시 시뮬레이션 모드 자동 폴백 (키 없어도 항상 동작).
- **화면 흐름**: `search → summary → nav → done` 4단계 상태머신, 마스터 시계 기반 보행자 시뮬레이션, 1×/2×/4× 배속.
- **접근성**: 라이트/야간/고대비 3중 테마, 고대비 모드 음성 강제 ON, 큰 글씨·터치 타일.

### 2.2 다듬을 갭
| 갭 | 현재 | 작업 |
|----|------|------|
| 라이브 데모 없음 | 로컬 실행만 가능 | GitHub Pages 배포 + 깔끔한 URL |
| 첫 로딩 빈 화면 | Babel 트랜스파일 중 공백 | 로딩 인디케이터 |
| 데모 모드 불명확 | 키 없으면 조용히 시뮬레이션 | "시뮬레이션 동작 중" 안내 배너 |
| 스크린샷 미노출 | `screenshots/` 19종이 repo에만 존재 | README에 대표 컷 임베드 |
| 23MB 단일 html 혼동 | 용도 불명확 | "오프라인 더블클릭용"으로 위치 정리 |

## 3. 작업 계획

### Phase 1 — 라이브 데모 배포 (핵심)
1. **루트 `index.html` 진입점 추가** — 소스 버전(`traffic_view/*.jsx`)을 로드해 `blackdew.github.io/klp-traffic-view/` 깔끔한 URL 제공.
2. **GitHub Pages 활성화** — `main` 브랜치 루트에서 서빙 (`gh` CLI 설정).
3. **로딩 인디케이터** — 트랜스파일 중 빈 화면 대신 간단한 스피너.
4. **데모 모드 배너** — API 키 없이 시뮬레이션으로 동작 중임을 사용자에게 안내.

### Phase 2 — README / 프레젠테이션 정비
5. **라이브 데모 배지/링크**를 README 최상단에 추가.
6. **대표 스크린샷 임베드** — 검색 / 요약 / 내비 / 접근성 4컷.
7. **23MB 단일 html 위치 정리** — "오프라인 더블클릭 실행용"으로 명확히 안내.

### Phase 3 — 소규모 UX 다듬기 (선택)
8. 모바일 메타·타이틀·파비콘 점검, 데모 진입 흐름 매끄럽게.
